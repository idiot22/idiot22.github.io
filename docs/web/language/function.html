    # 纯函数

    纯函数是这样一种函数，即相同的输入，永远会得到相同的输出，而且没有任何可观察的副作用

    函数的输出结果只和传入参数有关，函数不会去影响外面的值，外面的值也不会影响函数的结果

    # 纯函数的好处

    # 可缓存性

    要是相同的参数返回相同结果，就可以根据参数将结果缓存起来

    # 可测试性

    测试只要对函数添加相应的模拟值就可以测试了

    # 逻辑清晰

    从函数的参数就知道了它需要什么值，对于编程人员比较容易理解

    # 柯里化

    柯里化是一种将使用多个参数的一个函数转换成一系列使用一个参数的函数的技术
    curry 的这种用途可以理解为：参数复用。本质上是降低通用性，提高适用性

    // 1.
function add(a, b) {
    return a + b;
}

// 执行 add 函数，一次传入两个参数即可
add(1, 2) // 3

// 假设有一个 curry 函数可以做到柯里化
var addCurry = curry(add);
addCurry(1)(2) // 3

// 2.
var name = person.map(function (item) {
    return item.name;
})
// 优化
var prop = curry(function(key,obj){
  return obj[key]
})
var name = person.map(prop(name)) // 等价于 person.map(item => prop(name)(item))

    # 第一版

    function curry = function(fn){  // func1
  // arguments是类数组
  let args = [].slice.call(arguments, 1)
  return function(){ // func1
    let newArguments = [...args, ...arguments]
    return fn.apply(this, newArguments) // 相当于最后调用fn的参数是func1，func2两个参数和
  }
}
// 使用
function add(a, b) {
    return a + b;
}
// 作用就是可以把两个函数调用的参数都给传入的fn调用
var addCurry = curry(add, 1, 2);
addCurry() // 3
//或者
var addCurry = curry(add, 1);
addCurry(2) // 3
//或者
var addCurry = curry(add);
addCurry(1, 2) // 3

    # 第三版

    function curry(fn, args){
  let fnLength = fn.length
  args = args || []
  return function(){
    let allArgs = [...args, ...arguments]
    if(allArgs.length < fn.length){
      return curry.call(this, fn, allArgs)
    }else{
      fn.apply(this, allArgs)
    }
  }
}

    # 声明式代码

    声明式指出我们明确希望得到的结果(不太理解)
    声明式代码不指定执行顺序，所以它天然地适合进行并行运算

    • 不需要创建变量用来存储数据
    • 它不包含循环控制的代码如 for， while
    // 命令式
var makes = [];
for (i = 0; i < cars.length; i++) {
  makes.push(cars[i].make);
}
// 声明式
var makes = cars.map(function(car){ return car.make; });
