    # vue3项目结构

    # 与vue2的区别

    • vue3源码采用monorepo,将模块拆分到package目录中
    See More

    以前想用vue2中的某一模块，必须要改源码。现在可以单独引入那一个模块

    • vue3采用ts开发，vue2使用flow
    • vue3性能优化，支持tree-shaking，不使用就不会被打包
    • vue2后期引入RFC

    # 代码区别

    • vue3劫持数据采用proxy，vue2采用defineProperty，defineProperty有性能和缺陷，会递归，给属性增加get，set
    • vue3对模版编译进行了优化，编译时生成了block tree，可以对子节点的动态节点进行收集，可以减少比较，并且采用了patchFlag标记动态节点
    • vue3采用了compositionApi进行组织功能，解决了反复横跳，优化复用的逻辑（mxin带来的数据来源不清晰，命名冲突等），类型推断也更方便。vue2用的是options api
    • 增加了Fragment多个跟节点可以支持，Teleport，Suspense，keep-alive,transition
    • vue3diff算法（可以根据patchFlag做diff，采用了最长递增子序列）vue2全量diff
    • vue3 ts -> vue2 flow
    • vue3有自定义渲染器createRenderer()传入自己渲染的方法，可以根据vue革新来实现不同平台的代码
    • 模版编译的优化

    # createReactiveObject

    对外暴露就四个方法，比较显而易见，但里面的逻辑都可以抽象出一个公有的方法

    // 主要就是new一个代理，然后缓存代理，四个方法不同的地方都由他们的钩子完成
// 1. 判断对象， 2. 重复代理 3. 对不同类型进行proxy 4. 做缓存
export function createReactiveObject(target: any, isReadonly: boolean, baseHandlers: Object){
  // 如果目标不是对象，没法拦截，reactive这个api只能拦截对象类型
  if(!isObject(target)){
    return target
  }
  const proxyMap = isReadonly ? readonlyMap : reactiveMap
  let existProxy = proxyMap.get(target)
  if(existProxy){
    return existProxy
  }
  const proxy = new Proxy(target, baseHandlers)
  proxyMap.set(target, proxy)
  return proxy
}
