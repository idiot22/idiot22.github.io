    # 非父子之间的通讯

    1. 状态提升（中间人模式）
      主要用于兄弟组件，在父组件中存数据状态，一个组件改变了这个数据，然后父组件数据传入另一个组件，也就改变了
    2. 发布订阅者模式
      相当于vue中的eventBus
      主要就是搞清楚哪个是要传递参数，哪个是要接收参数，传递参数就是发布者（publish，eventBus.$on）,接收参数就是订阅者(subscribe,eventBus.$emit)
    // 最最简易版
const  bus = {
  list:[],
  // 相当于vue中的eventBus.$on
  subscribe(fn){
    this.list.push(fn)
  },
  // 相当于vue中的eventBus.$emit
  publish(val){
    this.list.forEach(fn=>{
      fn && fn(val)
    })
  }
}

    # 状态

    this.state 是纯 js 对象在 vue 中， data 属性是利用 Object.defineProperty 处理过的，更改 data 的数据的时候 会触发数据的 getter 和 setter ，但是 React 中没有做这样的处理，如果直接更改的话， react 是无法得知的，所以，需要使用特殊的更改状态的方法 setstate

    # setState

    # setState同步异步

    • setState存在同步逻辑中，异步更新状态，更新真实dom
    • setState存在异步逻辑，同步更新状态，同步更新dom
    • setState接收第二个参数，第二个参数式回调函数，状态，dom更新完就会被触发
    click(){
  // 只会执行一次，异步执行,在外部调用console，不会拿到最新的值，因为console会先执行，this.setState后执行
  // 第二个参数函数中写内容，才会是在setState调用完再调用
  this.setState({
    count: this.state.count++
  },()=>{
    console.log(this.state.count)
  })
  console.log(this.state.count)
  this.setState({
    count: this.state.count++
  })
  this.setState({
    count: this.state.count++
  })
  this.setState({
    count: this.state.count++
  })
}

    # useReducer

    let reducer = (prevState, action)=>{
  let newState = {...preState}

    case: 'change-name':
      newState.name = action.value
      return newState
    case: 'change-age',
      newState.age = action.value
      return newState
    default:
      return prevState
  }
