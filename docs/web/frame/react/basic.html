    # jsx语法

    将js，html混合的一种语法，将组件的结构，数据，样式都聚合在一起定义组件

    • jsx就是个对象，可以在if和for语句中使用jsx
    • 可以把jsx赋值给变量，还可以作为方法的参数，作为方法的返回值

    # React.createElement

    jsx转成createElement是在webpack编译的时候也就是打包的时候执行

    let element1 = <div id="qqq">hello world</div>
let element2 = React.createElement('div', {id: "qqq"}, "hello world")
// 最终解析成obj对象
let result = {
  type: 'div',
  props: {
    id: "qqq",
    children: "hello world"
  }

    # 事件

    不会真正的绑定事件到每一个具体的元素上，而是采用事件代理的模式

    1.不推荐
<button onClick={
  ()=>{
    console.log('事件方法')
  }
}>
2. this绑定不会有问题
<button onClick={this.handleClick.bind(this)}>
3. 用箭头函数定义，this也不会变，就是不好传参
<button onClick={this.handleClick}>
const handleClick = ()=>{
}
4. 推荐,方便传参，this指向还不会变
<button onClick={()=>this.handleClick()}></button>

    # 状态

    this.state 是纯 js 对象在 vue 中， data 属性是利用 Object.defineProperty 处理过的，更改 data 的数据的时候 会触发数据的 getter 和 setter ，但是 React 中没有做这样的处理，如果直接更改的话， react 是无法得知的，所以，需要使用特殊的更改状态的方法 setstate

    # setState

    state中定义了数据，不能直接去修改，修改了页面不会更新

    state = {
  aa: false
}
// 更新状态setStat
this.setState({
  aa: !this.state.aa
})

    # setState同步异步

    • setState存在同步逻辑中，异步更新状态，更新真实dom
    • setState存在异步逻辑，同步更新状态，同步更新dom
    • setState接收第二个参数，第二个参数式回调函数，状态，dom更新完就会被触发
    click(){
  // 只会执行一次，异步执行,在外部调用console，不会拿到最新的值，因为console会先执行，this.setState后执行
  // 第二个参数函数中写内容，才会是在setState调用完再调用
  this.setState({
    count: this.state.count++
  },()=>{
    console.log(this.state.count)
  })
  console.log(this.state.count)
  this.setState({
    count: this.state.count++
  })
  this.setState({
    count: this.state.count++
  })
  this.setState({
    count: this.state.count++
  })
}
