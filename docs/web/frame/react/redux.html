    # redux

    主要是用作应用状态的管理，redux用作一个单独的常量状态树保存这个页面的状态，这个对象不能直接被改变，当一些数据变化了，一个新的对象就会被创建，可以实现数据追踪，实现时光旅行

    # 设计

    1. state以单一对象存储在store对象中
    2. state只读
    3. 使用纯函数reducer执行state更新

    # 使用

    // store
import {createStore} from 'redux'
const reducer = (prevState, action)=>{
  let newState = {...prevState}
  switch(action.type){
    case 'show-tab-bar':
      newState.show = true
      return newState
    case 'hide-tab-bar':
      newState.show = false
      return newState
    default:
      return prevState
  }
}
const store = createStore(reducer)
export default store

    # 原理

    function store(reducer){
  let list = []
  let state = reducer()
  function subscribe(cb){
    list.push(cb)
  }
  function dispatch(action){
    state = reducer(state, action)
    for(let i in list){
      list[i] && list[i](action)
    }
  }
  function getState(){
  }
  return {
    subscribe,
    dispatch,
    getState
  }
}

    # redux中间件(处理异步)

    在redux里，action仅仅是携带了数据的普通js对象。 action creator返回的值是这个action类型的对象。然后通过store.dipatch进行分发。同步的情况下一切都很完美，但是reducer无法处理异步的情况。 那么我们就需要在action和reducer中间架起一座桥梁来处理异步。这就是middleware。

    • 中间件的由来与原理、机制
     export default function thunkMiddleware ({ dispatch , getState }){ 
   return next => action => typeof action == 'function' ? action (dispatch,getState )
 next ( action )}
    1
    2
    3

    这段代码的意思是，中间件这个桥梁接受到的参数 action ，如果不是 function,相当于中间件没有做任何事。如果 action 是 function ，则先执行 action , action 的处理结束之后，再在 action 的内部调用 dispatch。

    • 使用
     import reduxThunk from 'redux-thunk'
 import reduxPromise from 'redux-promise'
 const store = createStore(reducer, applyMiddleware(reduxThunk, reduxPromise))
 store.dispatch(getListAction())
 function getListAction(){
   return ({dispatch})=>{
     axios.get('/ddd').then((res)=>{
       dispatch({
         type: '33',
         value: res.data
       })
     })
   }
 }

    # react-redux

    redux和react其实没有关系，在上面的使用中需要去手动订阅值的变化，然后再去setState改变react中的值，还要取消订阅，就很麻烦。react-redux就对redux进行了封装，更加方便

    import { provider } from 'react-redux'
// 这样app下面的组件都可以接收到store了
<Provider store = { store}>
  <App></App>
</Provider>
