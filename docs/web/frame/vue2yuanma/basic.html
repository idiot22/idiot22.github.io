    # Vue 实例挂载的实现（$mount）

    1. 判断不能挂载到body和html上
    2. 如果没有定义render方法，则会把el或者template字符串转换成render方法

    在Vue2.0版本中，所有Vue的组件的渲染最终都需要render方法，无论我们是用单文件.vue方式开发组件，还是写了el或者template属性，最终都会转换成render方法

    1. 转成render函数后，就调用原型上的mount方法,实际上会去调用mountComponent
    2. mountComponent核心就是先调用vm._render方法先生成虚拟Node，再实例化一个渲染Watcher，在它的回调函数中会调用updateComponent方法，最终调用vm._upda更新DOM。

    在挂载的时候就会调用vm._render,_render在初始化的时候就已经vue原型加上了

    # 虚拟dom

    # 定义

    vnode.js

    一旦有更新创建就修改成千上百的dom，会变的很慢，就需要虚拟dom，虚拟dom可以比较比较新旧数据，找出修改的地方，更加快，虚拟dom实际就是用js对象来描述dom结构
    虚拟dom让组件的渲染逻辑与真实的dom操作解耦，并让它在其他环境中更直接地重用框架的运行

export function simpleNormalizeChildren (children: any) {
  for (let i = 0; i < children.length; i++) {
    if (Array.isArray(children[i])) {
      return Array.prototype.concat.apply([], children)
    }
  }
  return children

    createPatchFunction 内部定义了一系列的辅助方法，最终返回了一个 patch 方法

    通过 createPatchFunction 把差异化参数提前固化，这样不用每次调用 patch 的时候都传递 nodeOps 和 modules 了，这种编程技巧也非常值得学习。
    patch是跨端的，可以跑web和weex上，nodeOps，modules都是不同的
